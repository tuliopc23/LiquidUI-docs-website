# Inputs

Interactive form elements that provide a delightful and accessible way to gather user input and selections.

## Overview

Input components in Liquidify are crafted to follow Apple's Human Interface Guidelines, ensuring clarity, accessibility, and seamless user experiences. The components take advantage of advanced Liquid Glass effects to offer elegant, visually appealing interactions.

## Components

### [Glass Input](/components/inputs/glass-input)
A flexible text input element supporting various configurations.

- **Variants**: Regular, with icon, clear button
- **Sizes**: Small, Medium, Large
- **Features**: Placeholder, error messaging, validation

```tsx
import { GlassInput } from 'liquidify'

cGlassInput placeholder="Enter your name" size="md" /e
```

## Design Principles

### Apple Human Interface Guidelines Compliance

#### Clarity
- **Consistent styling**: Uniform visual language
- **Appropriate sizing**: Touch-optimized targets (minimum 44pt)

#### Deference
- **Content-focused**: Inputs complement content without overshadowing
- **Subtle animations**: Enhance without distracting

#### Depth
- **Layered interactions**: Visual hierarchy through elevation
- **Realistic motion**: Physics-based animations

### Liquid Glass Integration

All input components leverage Liquidify's advanced Liquid Glass material system:

- **Adaptive transparency**: Responsive to background content
- **Dynamic blur**: Creates depth without obscuring content  
- **Intelligent contrast**: Maintains readability in all contexts

### Accessibility Standards

#### VoiceOver Support
- **Descriptive labels**: Clear purpose and current state
- **State announcements**: Communicate changes to users

#### Keyboard Navigation
- **Tab navigation**: Full keyboard accessibility
- **Validation feedback**: Immediate error notifications
