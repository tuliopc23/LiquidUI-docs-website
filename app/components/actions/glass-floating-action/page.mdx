# Glass Floating Action

A prominent floating action button for primary actions that appears above the main content with smooth animations.

## Overview

The `GlassFloatingAction` component provides a Material Design-inspired floating action button with Liquidify's signature Liquid Glass effects. It's designed for the most important action on a screen and can expand to show text labels or additional options.

## Installation

```bash
npm install liquidify
```

## Basic Usage

```tsx
import { GlassFloatingAction } from 'liquidify'
import { Plus } from 'lucide-react'

function App() {
  return (
    <GlassFloatingAction 
      position="bottom-right"
      onClick={() => createNew()}
    >
      <Plus size={24} />
    </GlassFloatingAction>
  )
}
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `position` | `'top-left' \| 'top-right' \| 'bottom-left' \| 'bottom-right'` | `'bottom-right'` | Position on screen |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Button size |
| `extended` | `boolean` | `false` | Shows text label alongside icon |
| `variant` | `'primary' \| 'secondary'` | `'primary'` | Visual style variant |
| `disabled` | `boolean` | `false` | Disables interaction |
| `loading` | `boolean` | `false` | Shows loading spinner |
| `onClick` | `() => void` | - | Click handler |
| `children` | `ReactNode` | - | Icon and optional text content |
| `className` | `string` | - | Additional CSS classes |

### Size Specifications

| Size | Dimensions | Icon Size | Extended Width |
|------|------------|-----------|----------------|
| `sm` | 40×40px | 16px | Auto |
| `md` | 56×56px | 20px | Auto |
| `lg` | 72×72px | 24px | Auto |

## Usage Patterns

### Primary Actions
Use for the most important action on a screen:

```tsx
<GlassFloatingAction 
  position="bottom-right"
  onClick={() => createNewPost()}
>
  <Plus size={24} />
</GlassFloatingAction>
```

### Extended Labels
Show action text for clarity:

```tsx
<GlassFloatingAction 
  position="bottom-right"
  extended
  onClick={() => composeMessage()}
>
  <Edit size={24} />
  Compose
</GlassFloatingAction>
```

### Contextual Actions
Adapt to different screen contexts:

```tsx
// Chat screen
<GlassFloatingAction 
  position="bottom-right"
  extended
>
  <MessageCircle size={24} />
  New Chat
</GlassFloatingAction>

// Gallery screen
<GlassFloatingAction 
  position="bottom-right"
>
  <Camera size={24} />
</GlassFloatingAction>
```

## Accessibility

### VoiceOver Support
Proper labeling for screen readers:

```tsx
<GlassFloatingAction 
  aria-label="Create new post"
  position="bottom-right"
>
  <Plus size={24} />
</GlassFloatingAction>
```

### Keyboard Navigation
Full keyboard support:

- **Tab**: Focus the button
- **Enter/Space**: Activate action
- **Escape**: Dismiss any open menus

### Voice Control
Works with voice commands:

```tsx
// User can say: "Tap Create"
<GlassFloatingAction extended>
  <Plus size={24} />
  Create
</GlassFloatingAction>
```

## Animation & Performance

### Smooth Entrance
GSAP-powered entrance animations:

```tsx
<GlassFloatingAction 
  position="bottom-right"
  animateIn="scale"
>
  <Plus size={24} />
</GlassFloatingAction>
```

### Morphing States
Smooth transitions between states:

```tsx
const [extended, setExtended] = useState(false)

<GlassFloatingAction 
  extended={extended}
  onHover={() => setExtended(true)}
  onLeave={() => setExtended(false)}
>
  <Plus size={24} />
  {extended && 'Create New'}
</GlassFloatingAction>
```

## Best Practices

### Do's ✅

- **One per screen**: Use sparingly for primary actions
- **Clear purpose**: Make the action obvious and important
- **Consistent placement**: Keep position consistent across screens
- **Appropriate sizing**: Use larger sizes for more important actions

```tsx
// ✅ Good - Clear primary action
<GlassFloatingAction 
  position="bottom-right"
  aria-label="Add new contact"
>
  <Plus size={24} />
</GlassFloatingAction>
```

### Don'ts ❌

- **Multiple FABs**: Don't use more than one per screen
- **Secondary actions**: Don't use for less important actions
- **Blocking content**: Ensure it doesn't cover critical UI

```tsx
// ❌ Avoid - Multiple FABs on same screen
<GlassFloatingAction position="bottom-right">
  <Plus size={24} />
</GlassFloatingAction>
<GlassFloatingAction position="bottom-left">
  <Edit size={24} />
</GlassFloatingAction>
```
