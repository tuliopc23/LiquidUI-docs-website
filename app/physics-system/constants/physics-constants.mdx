# PHYSICS_CONSTANTS

The `PHYSICS_CONSTANTS` object provides fundamental physics values, mathematical constants, and configuration defaults used throughout the Liquidify physics system.

## Overview

`PHYSICS_CONSTANTS` centralizes all physics-related constants to ensure consistency across the animation system. These constants are carefully tuned for optimal visual results and realistic physics behavior.

## Import

```typescript
import { PHYSICS_CONSTANTS } from 'liquidify';
```

## Usage Examples

### Basic Physics Setup

```typescript
import { PHYSICS_CONSTANTS, SpringPhysics, Vector2D } from 'liquidify';

// Create spring with default constants
const spring = new SpringPhysics({
  stiffness: PHYSICS_CONSTANTS.DEFAULT_STIFFNESS,
  damping: PHYSICS_CONSTANTS.DEFAULT_DAMPING,
  mass: PHYSICS_CONSTANTS.DEFAULT_MASS,
});

// Apply gravity
const gravity = PHYSICS_CONSTANTS.GRAVITY;
```

### Particle System with Constants

```typescript
import { PHYSICS_CONSTANTS, ParticleEmitter } from 'liquidify';

const emitter = new ParticleEmitter({
  position: new Vector2D(100, 100),
  rate: PHYSICS_CONSTANTS.DEFAULT_EMISSION_RATE,
  maxParticles: PHYSICS_CONSTANTS.MAX_PARTICLES,
  particleLifetime: PHYSICS_CONSTANTS.DEFAULT_PARTICLE_LIFETIME,
  size: PHYSICS_CONSTANTS.DEFAULT_PARTICLE_SIZE,
  gravity: PHYSICS_CONSTANTS.GRAVITY,
  airResistance: PHYSICS_CONSTANTS.AIR_RESISTANCE,
});
```

### Animation Timing

```typescript
import { PHYSICS_CONSTANTS } from 'liquidify';

// Use consistent delta time
const deltaTime = PHYSICS_CONSTANTS.DEFAULT_DELTA_TIME;

// Clamp delta time for stability
function clampDeltaTime(dt: number): number {
  return Math.max(
    PHYSICS_CONSTANTS.MIN_DELTA_TIME,
    Math.min(PHYSICS_CONSTANTS.MAX_DELTA_TIME, dt)
  );
}
```

### Performance Optimization

```typescript
// Use constants for performance optimization
const usePerformanceConstants = () => {
  const maxParticles = PHYSICS_CONSTANTS.MAX_PARTICLES;
  const batchSize = PHYSICS_CONSTANTS.BATCH_SIZE;
  const cullingMargin = PHYSICS_CONSTANTS.CULLING_MARGIN;

  return {
    maxParticles,
    batchSize,
    cullingMargin,
    shouldCull: (
      position: Vector2D,
      viewport: { width: number; height: number }
    ) => {
      return (
        position.x < -cullingMargin ||
        position.x > viewport.width + cullingMargin ||
        position.y < -cullingMargin ||
        position.y > viewport.height + cullingMargin
      );
    },
  };
};
```

## Custom Constants

You can extend or override constants for specific use cases:

```typescript
// Create custom constants
const CUSTOM_PHYSICS_CONSTANTS = {
  ...PHYSICS_CONSTANTS,
  GRAVITY: new Vector2D(0, 15), // Stronger gravity
  DEFAULT_STIFFNESS: 150, // Stiffer springs
  PARTICLE_COLORS: ['#ff0000', '#00ff00', '#0000ff'], // Custom colors
};

// Use in your application
const spring = new SpringPhysics({
  stiffness: CUSTOM_PHYSICS_CONSTANTS.DEFAULT_STIFFNESS,
  damping: CUSTOM_PHYSICS_CONSTANTS.DEFAULT_DAMPING,
});
```

## Best Practices

1. **Use constants consistently** across your application
2. **Don't modify constants directly** - create custom versions instead
3. **Use appropriate gravity** for your scene's environment
4. **Clamp delta time** using MIN_DELTA_TIME and MAX_DELTA_TIME
5. **Respect performance limits** like MAX_PARTICLES
