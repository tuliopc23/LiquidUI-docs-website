# Design Tokens

Complete reference for Liquidify's design token system built on Apple's design principles and 4-point grid methodology.

## Overview

Design tokens are the visual design atoms of Liquidify — the smallest indivisible elements that represent design decisions. They create consistency, scalability, and maintainability across the entire design system.

## Import Tokens

```tsx
import { designTokens } from 'liquidify/tokens'
import type { 
  ColorToken, 
  SpacingToken, 
  TypographyToken,
  ComponentTokens 
} from 'liquidify/tokens'
```

## Spacing Tokens

Based on Apple's 4-point grid system for consistent layouts and optimal touch targets.

### Grid System
```tsx
const spacing = designTokens.spacing

// 4pt grid increments
spacing[0]  // 0px
spacing[1]  // 4px  - Smallest spacing
spacing[2]  // 8px  - Compact spacing
spacing[3]  // 12px - Close spacing
spacing[4]  // 16px - Default spacing
spacing[5]  // 20px - Comfortable spacing
spacing[6]  // 24px - Loose spacing
spacing[8]  // 32px - Section spacing
spacing[10] // 40px - Large spacing
spacing[12] // 48px - Extra large spacing
spacing[16] // 64px - Section dividers
spacing[20] // 80px - Page sections
spacing[24] // 96px - Major sections
spacing[32] // 128px - Hero sections
```

### Usage Examples
```tsx
// Component spacing
<GlassCard className="p-6 mb-8">  {/* 24px padding, 32px margin */}
  <h2 className="mb-4">Title</h2>   {/* 16px margin bottom */}
  <p className="mb-3">Content</p>   {/* 12px margin bottom */}
</GlassCard>

// Using token variables
<div style={{ padding: spacing[6], marginBottom: spacing[8] }}>
  Content with token-based spacing
</div>
```

### Usage Examples
```tsx
// Using typography tokens
<h1 style={{
  fontSize: fontSize.title1[0],
  lineHeight: fontSize.title1[1].lineHeight,
  letterSpacing: fontSize.title1[1].letterSpacing,
```

## Usage Best Practices

### 1. Use Semantic Tokens
```tsx
// ✅ Good - semantic meaning
<GlassButton variant="primary">Primary Action</GlassButton>

// ❌ Avoid - hard-coded values
<button style={{ backgroundColor: '#007AFF' }}>Button</button>
```

### 2. Leverage Component Tokens
```tsx
// ✅ Good - consistent sizing
<GlassButton size="lg">Touch-optimized</GlassButton>

// ❌ Avoid - custom sizing
<GlassButton style={{ height: '44px', padding: '14px 20px' }}>
  Custom Button
</GlassButton>
```

### 3. Responsive Typography
```tsx
// ✅ Good - responsive scale
<h1 className="hig-title-1 lg:hig-large-title">
  Responsive Heading
</h1>

// ✅ Good - programmatic sizing
<Text size={{ base: 'body', lg: 'title2' }}>
  Adaptive Text
</Text>
```

## CSS Custom Properties

All tokens are available as CSS custom properties:

```css
/* Spacing */
.component {
  padding: var(--spacing-4);
  margin-bottom: var(--spacing-6);
}

/* Typography */
.heading {
  font-family: var(--font-display);
  font-size: var(--font-size-title-1);
  font-weight: var(--font-weight-bold);
  line-height: var(--line-height-title);
}

/* Colors */
.button {
  background-color: var(--color-primary-500);
  border-color: var(--color-primary-600);
}

/* Glass materials */
.glass {
  background: var(--glass-light);
  backdrop-filter: blur(var(--glass-blur));
  border: 1px solid var(--glass-border-light);
}
```

## Platform Considerations

### iOS & iPadOS
- Tokens adapt to Dynamic Type scaling
- Respect user's preferred content size
- Support Smart Invert and high contrast modes

### macOS
- Optimized for pointer precision
- Supports system accent colors
- Adapts to window vibrancy settings

### watchOS  
- Reduced token set for efficiency
- Optimized for small screens
- Battery-conscious animations
