# Design Tokens

Complete reference for Liquidify's comprehensive design token system built on Apple's design principles and 4-point grid methodology.

## Overview

Design tokens are the visual design atoms of Liquidify — the smallest indivisible elements that represent design decisions. They create consistency, scalability, and maintainability across the entire design system.

## Import Tokens

```tsx
import {
  liquidGlassTokens,
  componentTokens,
  glassThemes,
  generateCSSCustomProperties,
} from 'liquidify';

import type {
  LiquidGlassTokens,
  ComponentTokens,
  GlassTheme,
  TokenValue,
} from 'liquidify';
```

## liquidGlassTokens

The core design token system providing comprehensive spacing, typography, colors, and animation values.

### Token Structure

```tsx
const liquidGlassTokens: LiquidGlassTokens = {
  spacing: SpacingTokens,
  typography: TypographyTokens,
  colors: ColorTokens,
  animation: AnimationTokens,
  glass: GlassTokens,
  breakpoints: BreakpointTokens,
  borderRadius: BorderRadiusTokens,
  shadows: ShadowTokens,
};
```

## Spacing Tokens

Based on Apple's 4-point grid system for consistent layouts and optimal touch targets.

### Grid System

```tsx
const spacing = liquidGlassTokens.spacing;

// 4pt grid increments
spacing[0]; // 0px
spacing[1]; // 4px   - Smallest spacing
spacing[2]; // 8px   - Compact spacing
spacing[3]; // 12px  - Close spacing
spacing[4]; // 16px  - Default spacing
spacing[5]; // 20px  - Comfortable spacing
spacing[6]; // 24px  - Loose spacing
spacing[8]; // 32px  - Section spacing
spacing[10]; // 40px  - Large spacing
spacing[12]; // 48px  - Extra large spacing
spacing[16]; // 64px  - Section dividers
spacing[20]; // 80px  - Page sections
spacing[24]; // 96px  - Major sections
spacing[32]; // 128px - Hero sections

// Semantic spacing
spacing.xs; // 4px   - Extra small
spacing.sm; // 8px   - Small
spacing.md; // 16px  - Medium (default)
spacing.lg; // 24px  - Large
spacing.xl; // 32px  - Extra large
spacing.xxl; // 48px  - Double extra large
spacing.xxxl; // 64px  - Triple extra large

// Touch targets
spacing.touchMin; // 44px - Minimum touch target
spacing.touchComf; // 48px - Comfortable touch target
spacing.touchLg; // 56px - Large touch target
```

### Usage Examples

```tsx
// Component spacing
<GlassCard className="p-6 mb-8">  {/* 24px padding, 32px margin */}
  <h2 className="mb-4">Title</h2>   {/* 16px margin bottom */}
  <p className="mb-3">Content</p>   {/* 12px margin bottom */}

// Using token variables
<div style={{ padding: spacing[6], marginBottom: spacing[8] }}>
  Content with token-based spacing
</div>

// Programmatic spacing

  padding: `${spacing[2]} ${spacing[4]}`,
```

## generateCSSCustomProperties

Utility function that converts design tokens into CSS custom properties for easy integration with CSS and styling frameworks.

### Basic Usage

```tsx
import { generateCSSCustomProperties, liquidGlassTokens } from 'liquidify';

// Generate CSS custom properties from tokens
const cssVariables = generateCSSCustomProperties(liquidGlassTokens);

// Apply to document root
const applyTokensToRoot = () => {
  const root = document.documentElement;
  Object.entries(cssVariables).forEach(([key, value]) => {
    root.style.setProperty(key, value);
  });
};

// Call on app initialization
applyTokensToRoot();
```

### Runtime Token Updates

```tsx
import { generateCSSCustomProperties } from 'liquidify';

// Update tokens dynamically
const updateTokens = newTokens => {
  const cssVars = generateCSSCustomProperties(newTokens);
  const root = document.documentElement;

  Object.entries(cssVars).forEach(([key, value]) => {
    root.style.setProperty(key, value);
  });
};

// Example: Update primary color
const updatePrimaryColor = color => {
  const updatedTokens = {
    ...liquidGlassTokens,
    colors: {
      ...liquidGlassTokens.colors,
      primary: color,
    },
  };
  updateTokens(updatedTokens);
};

// Usage
updatePrimaryColor('#FF6B6B');
```

## Usage Best Practices

### 1. Use Semantic Tokens

```tsx
// ✅ Good - semantic meaning
<GlassButton variant="primary">Primary Action</GlassButton>

// ❌ Avoid - hard-coded values
<button style={{ backgroundColor: '#007AFF' }}>Button</button>
```

### 2. Leverage Component Tokens

```tsx
// ✅ Good - consistent sizing
<GlassButton size="lg">Touch-optimized</GlassButton>

// ❌ Avoid - custom sizing
<GlassButton style={{ height: '44px', padding: '14px 20px' }}>

</GlassButton>
```

### 3. Responsive Typography

```tsx
// ✅ Good - responsive scale

```
