# Button

The Button component is a fundamental interactive element that triggers actions when clicked. It features beautiful glassmorphism effects and multiple variants to suit different use cases.

## Import

```jsx
import { Button } from 'glass-ui'
```

## Usage

### Basic Button

```jsx
<Button>Click me</Button>
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'primary' \| 'glass' \| 'outline' \| 'ghost'` | `'primary'` | Visual style variant |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Button size |
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `loading` | `boolean` | `false` | Whether the button is in loading state |
| `active` | `boolean` | `false` | Whether the button is in active state |
| `leftIcon` | `ReactNode` | - | Icon to display on the left |
| `rightIcon` | `ReactNode` | - | Icon to display on the right |
| `iconOnly` | `boolean` | `false` | Whether the button only contains an icon |
| `fullWidth` | `boolean` | `false` | Whether the button should take full width |
| `onClick` | `(event: MouseEvent) => void` | - | Click event handler |
| `type` | `'button' \| 'submit' \| 'reset'` | `'button'` | Button type |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Button content |

## Examples

### Form Submit Button

```jsx
function LoginForm() {
  const [loading, setLoading] = useState(false)
  
  const handleSubmit = async (e) => {
    e.preventDefault()
    setLoading(true)
    // Simulate API call
    await new Promise(resolve => setTimeout(resolve, 2000))
    setLoading(false)
  }
  
  return (
    <form onSubmit={handleSubmit}>
      <Input type="email" placeholder="Email" className="mb-4" />
      <Input type="password" placeholder="Password" className="mb-6" />
      <Button 
        type="submit" 
        variant="glass" 
        fullWidth 
        loading={loading}
      >
        {loading ? 'Signing in...' : 'Sign In'}
      </Button>
    </form>
  )
}
```

### Button Group

```jsx
function ButtonGroup() {
  const [selected, setSelected] = useState('option1')
  
  return (
    <div className="flex rounded-lg overflow-hidden border border-gray-300">
      <Button 
        variant={selected === 'option1' ? 'primary' : 'ghost'}
        onClick={() => setSelected('option1')}
        className="rounded-none"
      >
        Option 1
      </Button>
      <Button 
        variant={selected === 'option2' ? 'primary' : 'ghost'}
        onClick={() => setSelected('option2')}
        className="rounded-none border-l border-r"
      >
        Option 2
      </Button>
      <Button 
        variant={selected === 'option3' ? 'primary' : 'ghost'}
        onClick={() => setSelected('option3')}
        className="rounded-none"
      >
        Option 3
      </Button>
    </div>
  )
}
```

## Accessibility

The Button component follows WCAG 2.1 guidelines:

- **Keyboard Navigation**: Fully accessible via keyboard (Tab, Enter, Space)
- **Screen Readers**: Proper ARIA labels and roles
- **Focus Management**: Clear focus indicators
- **Color Contrast**: Meets AA contrast requirements
- **Loading States**: Announced to screen readers

<Callout type="info">
  When using `iconOnly` buttons, always provide an `aria-label` for screen readers.
</Callout>

### Accessibility Example

```jsx
<Button 
  iconOnly 
  variant="glass"
  aria-label="Close dialog"
  onClick={closeDialog}
>
  <CloseIcon />
</Button>
```

## Best Practices

1. **Use semantic button types** - Set `type="submit"` for form submissions
2. **Provide loading states** - Show feedback during async operations
3. **Use appropriate variants** - Primary for main actions, ghost for secondary
4. **Include icons thoughtfully** - Icons should enhance, not replace clear labels
5. **Consider mobile touch targets** - Ensure buttons are at least 44px tall
6. **Group related actions** - Use consistent styling for related buttons
