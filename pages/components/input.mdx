## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `"text" \| "email" \| "password" \| "number" \| "tel" \| "url"` | `"text"` | The HTML input type |
| `size` | `"sm" \| "default" \| "lg"` | `"default"` | The size of the input |
| `placeholder` | `string` | - | Placeholder text |
| `value` | `string` | - | The controlled value |
| `defaultValue` | `string` | - | The default value for uncontrolled usage |
| `disabled` | `boolean` | `false` | Whether the input is disabled |
| `required` | `boolean` | `false` | Whether the input is required |
| `className` | `string` | - | Additional CSS classes |
| `onChange` | `(event: ChangeEvent<HTMLInputElement>) => void` | - | Change event handler |
| `onFocus` | `(event: FocusEvent<HTMLInputElement>) => void` | - | Focus event handler |
| `onBlur` | `(event: FocusEvent<HTMLInputElement>) => void` | - | Blur event handler |

### Glassmorphism Properties

The GlassInput includes advanced visual effects:

- **Background Blur**: Backdrop filter with blur effect
- **Border Adaptation**: Dynamic border opacity based on background
- **Focus Enhancement**: Smooth focus transitions with ring effects
- **Content Awareness**: Adapts glass opacity to background content

### Accessibility

- Full keyboard navigation support
- Screen reader compatible with proper labeling
- Focus management and visual indicators
- High contrast mode support
- ARIA attributes for validation states

## Usage Guidelines

### Best Practices

- Always provide clear labels for accessibility
- Use appropriate input types for better mobile experience
- Provide helpful placeholder text that doesn't replace labels
- Include validation feedback for better user experience
- Group related inputs logically

### When to Use

- **Text**: General text input, names, titles
- **Email**: Email addresses with built-in validation
- **Password**: Sensitive data with masked characters
- **Search**: Search functionality with enhanced styling
- **Number**: Numeric data with appropriate keyboard on mobile

## Examples

### Controlled Input

```tsx
import { GlassInput } from "../../components/GlassComponents";
import { useState } from "react";

function ControlledInput() {
  const [value, setValue] = useState("");

  return (
    <GlassInput
      value={value}
      onChange={(e) => setValue(e.target.value)}
      placeholder="Type something..."
    />
  );
}
```

### With Validation

```tsx
import { GlassInput } from "../../components/GlassComponents";
import { useState } from "react";

function ValidatedInput() {
  const [email, setEmail] = useState("");
  const [error, setError] = useState("");

  const validateEmail = (value: string) => {
    const isValid = /\S+@\S+\.\S+/.test(value);
    setError(isValid ? "" : "Please enter a valid email");
  };

  return (
    <div>
      <GlassInput
        type="email"
        value={email}
        onChange={(e) => {
          setEmail(e.target.value);
          validateEmail(e.target.value);
        }}
        className={error ? "border-red-500" : ""}
        placeholder="Enter email"
      />
      {error && <p className="text-red-500 text-sm mt-1">{error}</p>}
    </div>
  );
}
```
