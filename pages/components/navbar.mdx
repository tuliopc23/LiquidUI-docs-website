## iOS Safe Area Support

The navbar automatically handles iOS safe areas (device notches, dynamic islands, etc.) using CSS environment variables:

- `env(safe-area-inset-top)` - Top safe area
- `env(safe-area-inset-left)` - Left safe area  
- `env(safe-area-inset-right)` - Right safe area

This ensures your navbar content is never hidden behind device UI elements, especially on newer iPhones.

```css
/* Automatically applied */
.glass-navbar {
  padding-top: env(safe-area-inset-top);
  padding-left: max(env(safe-area-inset-left), 1rem);
  padding-right: max(env(safe-area-inset-right), 1rem);
}
```

## API Reference

### GlassNavbar Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `brand` | `React.ReactNode` | - | Brand content (logo, text, etc.) |
| `links` | `NavLink[]` | `[]` | Navigation links array |
| `actions` | `React.ReactNode` | - | Action buttons or content |
| `position` | `"fixed" \| "sticky" \| "relative"` | `"fixed"` | Positioning behavior |
| `transparent` | `boolean` | `false` | Removes background for overlay use |
| `className` | `string` | - | Additional CSS classes |

### NavLink Interface

```typescript
interface NavLink {
  label: string;
  href: string;
  icon?: React.ReactNode;
}
```

## Best Practices

1. **Fixed Position**: Use for main site navigation that should always be accessible
2. **Sticky Position**: Use for section headers or contextual navigation
3. **Relative Position**: Use within containers or specific layouts
4. **Transparent**: Use over hero images or gradient backgrounds
5. **iOS Support**: No additional setup needed - safe areas are handled automatically
6. **Accessibility**: Always provide meaningful link labels and icons

## Mobile Considerations

On mobile devices, consider using the `GlassMobileNav` component for better touch interaction and space utilization. The navbar automatically hides links on smaller screens (using `hidden md:flex` classes).
