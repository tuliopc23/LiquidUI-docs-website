## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Card content |
| `asChild` | `boolean` | `false` | Render as a child element |

### Physics Properties

The GlassCard includes advanced glassmorphism features:

- **Adaptive Transparency**: Adjusts opacity based on background content
- **Depth Layering**: Creates visual hierarchy through multiple glass layers
- **Content Awareness**: Automatically adjusts blur and opacity for readability
- **Hover Enhancement**: Subtle interactive effects on hover

### Styling

Cards can be styled using Tailwind CSS classes:

```tsx
// Padding and spacing
<GlassCard className="p-4">   // Small padding
<GlassCard className="p-6">   // Medium padding (recommended)
<GlassCard className="p-8">   // Large padding

// Width constraints
<GlassCard className="max-w-sm">  // Small max width
<GlassCard className="max-w-md">  // Medium max width
<GlassCard className="max-w-lg">  // Large max width
<GlassCard className="w-full">    // Full width

// Height
<GlassCard className="h-32">      // Fixed height
<GlassCard className="min-h-40">  // Minimum height
<GlassCard className="h-full">    // Full height
```

## Accessibility

- Uses semantic HTML structure
- Proper focus management for interactive elements
- Screen reader compatible
- Keyboard navigation support
- High contrast mode support

## Best Practices

### Content Organization

- Use clear hierarchy with headings and sections
- Group related content together
- Provide adequate spacing between elements
- Use consistent padding throughout your application

### Visual Design

- Avoid nesting too many glass effects (max 2-3 levels)
- Ensure sufficient contrast for text readability
- Use consistent card sizes within the same context
- Consider responsive design for different screen sizes

### Performance

- Avoid excessive nesting of glass cards
- Use appropriate image optimization for card backgrounds
- Consider lazy loading for cards with heavy content
