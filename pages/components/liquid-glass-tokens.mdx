# Liquid Glass Tokens

Design tokens for the LiquidUI glassmorphism design system. These tokens provide consistent values for colors, spacing, blur effects, and opacity across all components.

## Overview

Liquid Glass Tokens are the foundational design constants that ensure visual consistency and systematic scaling across the entire LiquidUI component library. They define the core visual properties that create the signature glassmorphism aesthetic.

## Usage

### Importing Tokens

```typescript
import { 
  glassTokens, 
  colorTokens, 
  typographyTokens, 
  spacingTokens 
} from '@tuliocunha23/liquidui';
```

### Using in Components

```tsx
import { glassTokens, colorTokens } from '@tuliocunha23/liquidui';

function CustomGlassComponent() {
  return (
    <div className={`
      ${glassTokens.blur.md} 
      ${glassTokens.background.medium} 
      ${glassTokens.border.light}
      ${spacingTokens.borderRadius['2xl']}
      p-6
    `}>
      <h2 className="text-lg font-semibold">Custom Glass Component</h2>
      <p className="text-gray-600 mt-2">Using design tokens for consistency</p>
    </div>
  );
}
```

## Token Customization

### Extending Default Tokens

```typescript
import { glassTokens } from '@tuliocunha23/liquidui';

const customTokens = {
  ...glassTokens,
  blur: {
    ...glassTokens.blur,
    '4xl': 'backdrop-blur-4xl', // Custom blur level
    extreme: 'backdrop-blur-[100px]' // Arbitrary value
  },
  background: {
    ...glassTokens.background,
    minimal: 'bg-white/2', // Custom opacity
    maximum: 'bg-white/60'
  }
};
```

## Best Practices

### Consistency Guidelines

1. **Use Semantic Naming**: Choose tokens based on purpose, not appearance
2. **Maintain Hierarchy**: Follow the established scale for spacing and typography
3. **Test Accessibility**: Ensure sufficient contrast with glass effects
4. **Progressive Enhancement**: Design for opacity support fallbacks

### Performance Considerations

- Tokens are optimized for CSS-in-JS and utility-first frameworks
- Use CSS custom properties for runtime theme switching
- Leverage token tree-shaking for smaller bundle sizes
- Cache computed values for frequently used combinations

## Migration Guide

### From Custom Values

Before:
```css
.my-component {
  backdrop-filter: blur(12px);
  background: rgba(255, 255, 255, 0.15);
  border: 1px solid rgba(255, 255, 255, 0.25);
}
```

After:
```tsx
<div className={`${glassTokens.blur.md} ${glassTokens.background.light} ${glassTokens.border.light}`}>
  My Component
</div>
```

### From Other Design Systems

Map existing values to LiquidUI tokens:
```typescript
// Material-UI elevation -> Glass blur
const elevationToBlur = {
  1: glassTokens.blur.sm,
  2: glassTokens.blur.md,
  3: glassTokens.blur.lg,
  4: glassTokens.blur.xl
};
```
