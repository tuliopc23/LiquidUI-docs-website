## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string \| string[]` | - | The selected value(s) |
| `defaultValue` | `string \| string[]` | - | Default value for uncontrolled usage |
| `placeholder` | `string` | - | Placeholder text when no option is selected |
| `size` | `"sm" \| "default" \| "lg"` | `"default"` | Size of the select |
| `multiple` | `boolean` | `false` | Allow multiple selections |
| `searchable` | `boolean` | `false` | Enable search functionality |
| `searchPlaceholder` | `string` | `"Search..."` | Placeholder for search input |
| `disabled` | `boolean` | `false` | Whether the select is disabled |
| `required` | `boolean` | `false` | Whether selection is required |
| `clearable` | `boolean` | `false` | Show clear button when value is selected |
| `className` | `string` | - | Additional CSS classes |
| `onChange` | `(value: string \| string[]) => void` | - | Change event handler |
| `onSearch` | `(query: string) => void` | - | Search input change handler |
| `children` | `React.ReactNode` | - | Option elements |

### Advanced Features

- **Virtual Scrolling**: Efficient rendering for large option lists
- **Async Loading**: Support for dynamically loaded options
- **Custom Rendering**: Custom option and value rendering
- **Keyboard Navigation**: Full keyboard support with arrow keys
- **Accessibility**: ARIA compliance and screen reader support

### Accessibility

- **Keyboard Navigation**: Arrow keys, Enter, Escape, and Tab support
- **Screen Readers**: Proper ARIA attributes and announcements
- **Focus Management**: Clear focus indicators and states
- **High Contrast**: Respects system high contrast settings

## Usage Guidelines

### Best Practices

- Provide clear, descriptive labels for all selects
- Use appropriate placeholder text
- Group related options when possible
- Consider searchable select for 10+ options
- Use multi-select sparingly to avoid overwhelming users

### When to Use

- **Single Selection**: Choosing one option from a list
- **Multiple Selection**: Selecting multiple related items
- **Searchable**: Large lists that benefit from filtering
- **Categorized**: Options that can be logically grouped

## Examples

### Controlled Select

```tsx
import { GlassSelect } from "../../components/GlassComponents";
import { useState } from "react";

function ControlledSelect() {
  const [value, setValue] = useState("");

  return (
    <GlassSelect
      value={value}
      onChange={setValue}
      placeholder="Choose an option"
    >
      <option value="option1">Option 1</option>
      <option value="option2">Option 2</option>
      <option value="option3">Option 3</option>
    </GlassSelect>
  );
}
```

### Multi-Select with Tags

```tsx
import { GlassSelect } from "../../components/GlassComponents";
import { useState } from "react";

function MultiSelectExample() {
  const [selectedValues, setSelectedValues] = useState([]);

  const handleRemove = (valueToRemove) => {
    setSelectedValues(prev => 
      prev.filter(value => value !== valueToRemove)
    );
  };

  return (
    <div>
      <GlassSelect
        multiple
        value={selectedValues}
        onChange={setSelectedValues}
        placeholder="Select multiple options"
      >
        <option value="react">React</option>
        <option value="vue">Vue</option>
        <option value="angular">Angular</option>
      </GlassSelect>
      
      <div className="flex flex-wrap gap-2 mt-2">
        {selectedValues.map(value => (
          <span 
            key={value} 
            className="glass-effect px-2 py-1 rounded flex items-center gap-2"
          >
            {value}
            <button onClick={() => handleRemove(value)}>Ã—</button>
          </span>
        ))}
      </div>
    </div>
  );
}
```
